§ III

Modeling pipeline

Split first; fit preprocessing, resampling, and the estimator inside each CV fold. Everything downstream inherits the same imblearn.Pipeline so the model ships with its own preprocessing.

01
Ingest
Kaggle public endpoint → data/raw/ → stratified 80/20 split persisted as Parquet.
02
Preprocess
ColumnTransformer: median + scale for numerics, mode + one-hot for categoricals.
03
Resample
SMOTE (k=3) applied inside each CV fold — no leakage.
04
Benchmark
5 candidates × 5-fold stratified CV on average_precision, logged to MLflow.
05
Tune
Optuna TPE on the winner, 30 trials, same CV folds.
06
Evaluate
Held-out test: threshold pick, diagnostics, model card.

Why average_precision and not accuracy? At a 2.58 % positive rate, a model that always predicts 0 already scores ~97 % accuracy. PR-AUC directly rewards ranking the 31 positives above the 1 169 negatives — which is the only thing that matters here.

A word of honesty about perfect scores

This dataset appears to be near-deterministically separable: once stress_level, anxiety_level and addiction_level are simultaneously high and sleep_hours is low, the label is essentially fixed. The tuned model learns that boundary cleanly and recovers every test positive.

Perfect CV and test metrics are therefore a property of the dataset, not a license to deploy this for clinical use. The full caveat list is in MODEL_CARD.md.

§ VII

Model card

The short version. MODEL_CARD.md in the repo has the full classification report and training configuration.

Intended use

Educational and benchmarking only. This repo demonstrates a full ML engineering workflow (ingest → EDA → preprocess → benchmark → tune → evaluate → persist) on an imbalanced tabular dataset. Not for clinical decision-making.

Limitations

  • Target is extremely imbalanced (~2.6 %). PR-AUC and F1 are informative; raw accuracy is not.
  • The dataset is synthetic / self-reported.
  • Permutation importance reflects predictive signal on this dataset, not causality.
  • Generalisation outside the demographic/age range (13–19) is untested.

Training setup

  • CV: StratifiedKFold(5), seed 42
  • Scoring: average_precision
  • Imbalance: SMOTE (k_neighbors=3) inside the fold
  • Winner: GradientBoostingClassifier, Optuna-tuned
§ VIII

Reproducibility

Three commands on any machine with uv. The pipeline is deterministic under the default seed — you should get the same benchmark scores, the same tuned params, and the same figures. 

# 1. Install uv (Astral)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. Clone + install deps
git clone https://github.com/DeryFerd/Test-Devin-Opus-4.7-ML.git
cd Test-Devin-Opus-4.7-ML
uv sync --extra dev

# 3. Run the full pipeline (ingest → eda → train → tune → evaluate)
uv run teen-mh all

# Or run stage-by-stage
uv run teen-mh ingest
uv run teen-mh eda
uv run teen-mh train
uv run teen-mh tune
uv run teen-mh evaluate

# Browse MLflow
uv run mlflow ui   # http://127.0.0.1:5000
